# 勤怠イレギュラーメモ

勤怠の入力漏れを防止するためのスマートフォン向けWebアプリ。
月末にまとめて勤怠システムへ入力する際の作業漏れをチェックリストで管理する。

## ファイル構成

```
index.html   # アプリ本体（単一HTMLファイル）
```

ビルドシステム・外部依存なし。バニラJS + インラインCSS の単一ファイル構成。

## 技術スタック

- HTML / CSS / Vanilla JavaScript（ES5）
- フォント：Hiragino Sans / Meiryo
- データ永続化：なし（メモリのみ）→ テキストコピーでiPhoneメモアプリに保存し、次回「メモから読み込む」で復元

## 主な機能

### 人切り替え
3人（自分 / Hさん / Kさん）を切り替えて個別に記録を管理。
データは `allEntries = { self: [], h: [], k: [] }` に格納。

### イレギュラー種別

| キー | 名称 | 月末チェック対象 |
|------|------|-----------------|
| `remote` | 在宅切り替え | 現場システム・現場Excel・HRMOS の休憩時間修正 |
| `training` | 研修・社内イベント | HRMOS に研修時間を加算 |
| `holiday` | 休暇取得 | 現場Excel に休暇種別記入・HRMOS に休暇申請 |
| `late` | 遅刻・早退 | 現場システム・現場Excel・HRMOS・自社システムの時刻修正 |
| `bukai` | 部会 | HRMOS に 19:00〜21:00 を残業登録 |

### チェックリスト
各エントリには月末入力作業のタスクが紐づく（`TASKS` オブジェクトで定義）。
完了済みにチェックを入れると進捗表示が更新される。

### エクスポート／インポート
- **エクスポート**：当月分を `【種別】M月D日（曜） 詳細` 形式でテキスト化 → コピーモーダルで表示
- **インポート**：上記形式のテキストを貼り付けて読み込む（重複チェックあり）

## 主要データ構造

```js
// エントリ
{
  id: Date.now(),        // 一意ID
  type: 'remote',        // イレギュラー種別
  date: '2026-05-18',    // ISO日付
  detail: '休憩60分',    // 入力詳細テキスト
  tasks: [false, false]  // チェックリストの完了状態
}
```

## 開発上の注意

- **データ永続化なし**：`localStorage` 未使用。ページを閉じるとデータが消えるため、登録後に必ずコピーモーダルでテキストを保存する仕様。
- **ES5記法**：`var`・`function`宣言・コールバック形式で統一。`const`/`let`/アロー関数は使わない。
- **モバイル優先**：`max-width: 480px` で中央揃え。`font-size: 16px` でiOSのズームを防止。
- **CSSカスタムプロパティ**：カラーパレットは `:root` の CSS変数で一元管理。
